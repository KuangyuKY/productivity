# 1701.csv 原始发票数据检查说明

## 1. 检查目的

本次检查的目标是确认 `1701.csv` 是否为 3,410 家样本企业对应的原始发票明细数据。

用于核对的样本企业名单来自 `firm_city.csv`，其中 `企业ID` 去重后共有 3,410 个，与本次研究样本企业数量一致。

## 2. 文件基本情况

`1701.csv` 是一个较大的原始发票级 CSV 文件，文件大小约 39.8GB。

文件表头如下：

```text
开票日期, 购方企业ID, 购方地区, 销方企业ID, 销方地区, 项目代码, 项目, 开票金额, 单位, 数量, 单价, 税额
```

从字段结构看，该文件是发票明细级数据，而不是企业、城市或产品层面的聚合数据。

## 3. 判断标准

如果一条发票记录满足以下任一条件，就认为它与 3,410 家样本企业相关：

1. `购方企业ID` 属于 `firm_city.csv` 中的 3,410 个样本企业 ID；
2. `销方企业ID` 属于 `firm_city.csv` 中的 3,410 个样本企业 ID。

因此，本次检查的核心判断是：

```text
购方企业ID in 样本企业列表 OR 销方企业ID in 样本企业列表
```

## 4. 检查方法

由于 `1701.csv` 文件很大，不能直接一次性读入内存，因此采用流式扫描方法进行检查。

检查过程包括三步：

1. 检查文件表头和前几行，确认字段结构；
2. 对前 1,000,000 行进行样本企业匹配检查；
3. 对全文件进行流式 CSV 解析，逐行判断购方或销方是否属于样本企业。

另外，也对文件不同位置的 chunk 做了分散抽样检查，包括文件开头、10%、25%、50%、75%、90% 附近的位置。

## 5. 全量检查结果

稳健 CSV 解析器的全量扫描结果保存在：

```text
productivity/1701_sample_check_summary.txt
```

主要结果如下：

| 指标 | 数值 |
|---|---:|
| 总发票记录数 | 275,077,645 |
| 样本企业数量 | 3,410 |
| 购方属于样本企业的记录数 | 166,133,198 |
| 销方属于样本企业的记录数 | 209,568,604 |
| 购方或销方至少一方属于样本企业的记录数 | 275,077,645 |
| 购销双方都属于样本企业的记录数 | 100,624,157 |
| 购销双方都不属于样本企业的记录数 | 0 |
| 购方或销方至少一方属于样本企业的比例 | 1.00000000 |
| 购销双方都不属于样本企业的比例 | 0.00000000 |

## 6. 关于快速扫描结果的说明

辅助检查中还运行了一个快速扫描脚本，结果保存在：

```text
productivity/1701_fast_sample_check_summary.txt
```

快速扫描显示有 120 行似乎购销双方都不在样本企业中。但这个结果不是最终判断依据，原因是快速扫描使用了简化的逐行分割方法，在遇到字段内换行或复杂引号时，会把同一条 CSV 记录误拆成多行。

这一点可以从结果中看出：

1. 快速扫描的总行数是 275,077,765，比稳健 CSV 解析器多 120 行；
2. 快速扫描记录的第一条异常行是残缺片段，不是完整发票记录。

因此，最终应以稳健 CSV 解析器的结果为准。

## 7. 最终判断

`1701.csv` 可以确认为 3,410 家样本企业对应的原始发票明细数据。

它的筛选口径不是“样本企业只作为购方”，也不是“样本企业只作为销方”，而是：

```text
每条发票记录中，购方或销方至少一方属于 3,410 家样本企业。
```

换言之，`1701.csv` 是围绕这 3,410 家企业展开的交易网络型原始发票数据，既包括样本企业作为购买方的交易，也包括样本企业作为销售方的交易。
