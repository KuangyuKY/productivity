# 2017年发票数据 — 数据说明(供 Python / Stata 分析使用)

## 概览

本数据为 2017 年的增值税发票流水,已从数据库抽样并导出为 **CSV 文件**,后续分析在 Python(pandas)或 Stata 中进行。

全年数据分为三个**结构完全一致**的文件,合并(pandas `concat` / Stata `append`)后即为完整的 2017 年样本数据:

| 文件 | 内容 |
|---|---|
| `sample_GX1701.csv` | 2017 年发票记录的一部分 |
| `sample_GX1702.csv` | 2017 年发票记录的一部分 |
| `sample_GX1703.csv` | 2017 年发票记录的一部分 |

每一行代表一条发票明细:某个**购方企业**向某个**销方企业**就某个**项目(商品/服务)**开具的一笔发票。

**抽样方式**:按企业抽样。先随机选定一批样本企业(基于企业 ID),再取这些企业作为购方或销方的**全部**发票记录。因此被选中的企业,其全年买卖记录是完整的;但样本企业偏向交易量较大的企业(抽样机制导致),分析时需注意这一选择性。

## 读入与清洗的关键注意点

### 1. `数量` 是文本,需手动转数值
原始数据中 `数量` 以字符串存储(内容看似数字:`1`、`193`、`-1`、`1000`)。直接做算术或聚合前必须转换,并处理缺失值与潜在非数字字符。

- **Python**:`df['数量'] = pd.to_numeric(df['数量'], errors='coerce')`(无法转换的变 `NaN`)
- **Stata**:`destring 数量, generate(数量_num) force`(`force` 把非法值置为缺失)

### 2. 长 ID / 代码必须当字符串,防止精度丢失
`项目代码` 是 19 位长数字串(如 `3040502020400000000`)。若被当成数值读入,会超出常规整数/浮点精度而被截断或变成科学计数法,导致代码错乱。`购方企业ID`、`销方企业ID` 虽是普通整数,但既然只作标识符、不参与运算,也建议按字符串处理更安全。

- **Python**:读入时指定 `dtype`,例如
  ```python
  df = pd.read_csv('sample_GX1701.csv',
                   dtype={'项目代码': str, '购方企业ID': str, '销方企业ID': str,
                          '购方地区': str, '销方地区': str},
                   encoding='utf-8')
  ```
- **Stata**:`import delimited` 时对这些列用 `stringcols()` 指定,或读入后保持字符串,不要 `destring`。

### 3. 编码为 UTF-8(中文不乱码)
`项目`、`单位` 等为中文。读入时显式指定 UTF-8:
- **Python**:`encoding='utf-8'`(若仍异常可试 `utf-8-sig`)
- **Stata**:`import delimited ..., encoding("UTF-8")`(Stata 14+ 默认 UTF-8;若乱码可指定 `encoding("utf-8")` 或先用 `unicode` 工具转换)

### 4. 金额/税额的负值是正常业务,勿误删
`开票金额`、`税额` 存在大量负值(如 `-10089.78`),对应发票**红冲、退货、折让**等。这是真实业务记录,**不应**作为异常值直接剔除;是否纳入或单独处理取决于你的分析口径(如计算净额时需保留,统计开票笔数时需考虑红冲与原票配对)。

### 5. 缺失值
`单位`、`数量` 缺失较多;导出的 CSV 中缺失可能呈现为空字符串或 `NULL`/`(Null)` 文本。读入时统一识别为缺失:
- **Python**:`pd.read_csv(..., na_values=['', 'NULL', '(Null)'])`
- **Stata**:`destring`/`replace` 时把 `"NULL"`、`""` 处理为 `.`

### 6. 同一企业可同时为购方与销方
一家企业的"全部业务"需同时看它出现在 `购方企业ID` 和 `销方企业ID` 两个角色下的记录。数据中也存在 `购方企业ID == 销方企业ID` 的自交易记录。若要构建企业层面的汇总(如每家企业的总销售/总采购),需分别按两个 ID 列聚合。

### 7. 三文件合并即全年
三个 CSV 列完全一致,合并即 2017 全年样本。建议合并后再增加一个来源标记列(记录来自 1701/1702/1703),以便需要时追溯。
